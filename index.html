Tích hợp tổng đài Stringee để AI tự động gọi điện ra số di động thật và đàm thoại trực tiếp với khách hàng.

Mã nguồn mẫu kết nối Backend (Node.js)

server.js (zca-js Integration) 
const { ZcaClient } = require('zca-js');
const express = require('express');
const axios = require('axios');
const app = express();

app.use(express.json());

// Khởi tạo ZcaClient giả lập Zalo Web
const client = new ZcaClient({
    sessionPath: './zalo_session',
    headless: true
});

client.on('message', async (message) => {
    console.log(`Nhận tin nhắn từ ${message.senderId}: ${message.body}`);
    
    // Gửi tin nhắn sang Dashboard thông qua Webhook
    await axios.post('http://localhost:8080/api/webhook', {
        event: 'message.received',
        data: message
    }).catch(err => console.log('Lỗi gửi Webhook:', err.message));
});

// Endpoint Dashboard gọi xuống để điều khiển Zalo
app.post('/api/send-message', async (req, res) => {
    const { groupId, text } = req.body;
    try {
        await client.sendMessage(groupId, text);
        res.json({ success: true, message: 'Đã gửi thành công!' });
    } catch (error) {
        res.status(500).json({ success: false, error: error.message });
    }
});

client.login().then(() => {
    app.listen(3000, () => console.log('Zalo API Gateway chạy ở port 3000'));
});

  • Giãn cách tin nhắn tối thiểu 5s.
  • Sử dụng SpinTax (nội dung xoay vòng tránh trùng lặp).
  • Giới hạn gửi tối đa 100 nhóm/ngày/tài khoản.
  • Không sử dụng số điện thoại chính làm bot đại trà.
  • Tránh spam liên tục các từ khóa tài chính nhạy cảm.

Khuyến Nghị An Toàn Khi Sử Dụng Zalo Automation

Hệ thống Zalo kiểm soát rất chặt chẽ hành vi gửi tin nhắn tự động hàng loạt bằng thuật toán AI phát hiện bất thường. Để tránh việc tài khoản Zalo cá nhân của bạn bị block hoặc khóa số điện thoại vĩnh viễn, vui lòng tuân thủ các nguyên tắc sau:

1. Tránh gửi nội dung trùng lặp (Spamming)

Luôn thiết lập các biến nội dung ngẫu nhiên (SpinTax) để mỗi tin nhắn gửi đi đều khác biệt. Hệ thống Zalo sẽ quét và phát hiện nếu hàng chục tin nhắn giống hệt nhau được gửi đi trong vài phút.

2. Sử dụng thời gian trễ ngẫu nhiên (Randomized Delay)

Tuyệt đối không gửi tin nhắn với khoảng cách đều tăm tắp (ví dụ: cứ 3.0s một tin). Hãy bật Safe Mode với khoảng cách chờ ngẫu nhiên từ 5s - 15s để giả lập hành vi gõ bàn phím và gửi tay của người dùng thật.

3. Phân chia tệp gửi và giới hạn tài khoản

Nếu bạn quản lý hàng chục hoặc hàng trăm nhóm, đừng bao giờ dùng 1 tài khoản để gửi cho toàn bộ. Hãy chia đều cho các tài khoản phụ (Clone) và giới hạn mỗi tài khoản chỉ nên gửi tối đa 40 - 50 nhóm mỗi ngày.

⚠️ Chú ý về số điện thoại

Nên sử dụng các số điện thoại phụ đã được xác thực (qua căn cước công dân nếu cần) để chạy các tính năng tự động. Không nên dùng số Zalo chính chứa danh bạ khách hàng quan trọng để giảm thiểu rủi ro mất tài khoản liên lạc chính.
